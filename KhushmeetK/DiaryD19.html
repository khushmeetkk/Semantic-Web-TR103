Day 19 - GitHub and Collaborative Development

GitHub

GitHub is a web-based platform used for version control and collaborative software development. It allows developers to store, track, and manage their code using Git, a distributed version control system. GitHub provides a user-friendly interface for working with Git repositories, enabling collaboration among multiple contributors on the same project.

Collaborative Development with Git

Collaborative development is essential for teams working on projects together. GitHub facilitates this by allowing multiple developers to work on different branches of a project simultaneously. Key features that support collaborative development include:

  • Pull Requests: A method for proposing changes to a repository. Other team members can review, comment on, and merge the changes into the main branch.
  • Issues: A way to track bugs, enhancements, and tasks within a project. Issues can be assigned to specific team members, improving project management.
  • Project Boards: Kanban-style boards that help teams organize and prioritize work visually.

Getting Started with GitHub Desktop and Command Line

GitHub Desktop provides a graphical interface for Git and GitHub, simplifying many tasks for those who prefer not to use the command line. It allows you to:

  • Clone repositories from GitHub to your local machine.
  • Stage changes, commit them, and push to the remote repository.
  • Switch between branches and manage merge conflicts visually.

Using Git from the Command Line

Many developers prefer using Git via the command line for greater control and flexibility. Here are some essential commands:

git clone    # Clones a repository to your local machine
git status                    # Displays the status of your working directory
git add                 # Stages changes for commit
git commit -m "message"       # Commits staged changes with a message
git push                       # Pushes changes to the remote repository
git pull                       # Fetches changes from the remote repository

Authentication with TOTP

TOTP (Time-based One-Time Password) is a secure method for two-factor authentication (2FA) that adds an extra layer of security to your GitHub account. When enabled, TOTP requires a code generated by an authentication app, such as Google Authenticator or Authy, in addition to your password. This helps protect your account from unauthorized access.

Setting Up TOTP for GitHub

  1. Go to your GitHub account settings.
  2. Select "Security" from the sidebar.
  3. Click on "Enable two-factor authentication".
  4. Choose the "Set up using an app" option.
  5. Scan the QR code with your authentication app to generate your TOTP codes.
  6. Enter the generated code on GitHub to confirm.
