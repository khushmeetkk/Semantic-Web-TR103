Day 9 - Requirements and Tools for Web Optimizatio

FR & NFR

We delved into the concepts of functional and non-functional requirements, which are crucial for the successful development of any web application:

  • Functional Requirements: These are the specific behaviors and functionalities that a system must exhibit. They describe what the system should do, including features, operations, and interactions.
  • Non-Functional Requirements: These requirements specify how a system performs its functions. They include aspects such as performance, usability, reliability, and security. Non-functional requirements ensure that the system is user-friendly and meets various quality standards.

Tools for Web Optimization

We also explored various tools for web optimization that help enhance the performance and efficiency of web applications:

  • Lighthouse: An open-source tool for auditing the performance, accessibility, and SEO of web pages. It provides actionable insights to improve the overall quality of the site.
  • GTmetrix: A tool that analyzes the speed of your web pages and offers suggestions for optimization based on various performance metrics.
  • CDN (Content Delivery Network): A network of servers distributed geographically to deliver web content to users more efficiently by reducing latency and load times.
  • Google PageSpeed Insights: This tool evaluates the performance of a web page on both mobile and desktop devices, providing suggestions for optimizing load times and overall user experience.
  • Image Optimization: Techniques used to reduce the file size of images without compromising quality, enhancing load times and performance.
  • NVDA Screen Reader: A free and open-source screen reader that allows visually impaired users to interact with web content, highlighting the importance of accessibility.
  • Minification: The process of removing unnecessary characters from CSS and JavaScript files to reduce their size and improve load times.

Practical

In our practical session, we utilized several of the above tools to optimize our web project. The focus was on:

  • Performance Auditing: Using Lighthouse to evaluate our website's performance and identify areas for improvement.
  • Speed Analysis: Leveraging GTmetrix and Google PageSpeed Insights to analyze load times and get optimization recommendations.
  • Image Optimization: Implementing techniques to compress images on our web pages, ensuring faster load times without sacrificing quality.
  • Accessibility Testing: Using the NVDA screen reader to ensure our web content is accessible to all users, highlighting areas where improvements were needed.

Key Takeaways

Through this practical experience, we gained valuable insights into how web optimization tools can significantly enhance the performance and usability of our web projects. Understanding both functional and non-functional requirements is essential in delivering high-quality applications that meet user needs effectively. We emphasized the importance of continuous optimization and user-centric design in web development.
