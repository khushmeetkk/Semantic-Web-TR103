Day 10 - Browser Extensions

Learnings

On the tenth day, we focused on various browser extensions that can significantly enhance our web development and testing capabilities. These tools help in improving website quality, understanding web technologies, and enhancing user experience. Here’s a brief overview of the extensions we explored:

  • HTML Validator: This extension checks the HTML code of web pages for errors, helping developers ensure their markup is compliant with web standards.
  • Wappalyzer: A tool that detects technologies used on websites, such as content management systems, eCommerce platforms, and analytics tools, providing insights into web technology stacks.
  • ColorZilla: A color picker and eyedropper tool that allows you to sample colors from any web page, aiding in web design and ensuring consistent color schemes.
  • WhatFont: This extension helps you identify the fonts used on web pages simply by hovering over the text, making it easy to replicate styles or learn about typography choices.
  • WAVE: A web accessibility evaluation tool that helps developers identify accessibility issues on their websites, ensuring compliance with accessibility standards and improving usability for all users.

Practical

In our practical session, we installed and utilized the mentioned browser extensions to enhance our web development process. Here’s a summary of our tasks:

Installation of Extensions

We started by installing each of the listed extensions from the Chrome Web Store. This involved:

  • Searching for the extensions in the Chrome Web Store.
  • Clicking the "Add to Chrome" button for each extension.
  • Granting the necessary permissions to enable their functionalities.

Using the Extensions

Once installed, we practiced using each extension:

  • HTML Validator: We tested a sample web page and identified several HTML validation errors, helping us understand the importance of clean code.
  • Wappalyzer: By visiting different websites, we analyzed the technologies powering them, gaining insights into popular frameworks and tools used across the web.
  • ColorZilla: We sampled colors from a website and created a color palette to use in our own web projects, ensuring cohesive design.
  • WhatFont: We explored various web pages to identify fonts, noting their sizes and styles for future reference in our designs.
  • WAVE: We ran accessibility checks on our practice website, learning how to address accessibility issues effectively.
